# govhrapp

A set of interactive Shiny applications to clean and analyze government human resources personnel data. The package provides two application suites:

- **Quality Control** (`run_govhrapp()`): tools for detecting data quality issues — missingness, volatility, and basic data diagnostics.
- **Analytics** (`run_wagebillapp()`, `run_workforceapp()`): interactive dashboards for wage bill and workforce headcount analysis.

---

## Installation

The package depends on the internal `govhr` package. Install both from GitHub:

```r
# Install govhr first
remotes::install_github("WB-PIDA-Data-Science-Shop/govhr")

# Install govhrapp
remotes::install_github("WB-PIDA-Data-Science-Shop/govhrapp")
```

---

## Running locally

```r
pkgload::load_all(".")

# Quality control suite
run_govhrapp()

# Wage bill analytics
run_wagebillapp(wagebill_data = your_data)

# Workforce analytics
run_workforceapp(workforce_data = your_data)
```

Alternatively, `app.R` at the project root loads and launches `run_govhrapp()` directly and can be used with the RStudio/Positron "Run App" button.

## Deployment to Posit Connect

Deployment is managed via `deploy_govhrapp()`. Each application suite has a registered GUID on Posit Connect stored as an environment variable.

### 1. Set environment variables

Add the following to your `.Renviron` file (run `usethis::edit_r_environ()` to open it):

```
govhrapp_quality_guid=<your-quality-app-guid>
govhrapp_analytics_guid=<your-analytics-app-guid>
```

### 2. Configure Posit Connect credentials

If not already set up:

```r
rsconnect::setAccountInfo(
  name   = "<account-name>",
  token  = "<token>",
  secret = "<secret>"
)
```

### 3. Deploy

```r
pkgload::load_all(".")

# Deploy quality control suite
deploy_govhrapp("quality")

# Deploy analytics suite
deploy_govhrapp("analytics")
```

This calls `rsconnect::deployApp(appDir = ".", appId = <guid>, forceUpdate = TRUE)`, pushing the full package directory to Connect and updating the existing deployment in place.

### Notes

- The `renv.lock` file ensures the Connect server installs the same package versions used locally. Posit Connect must be able to reach the `govhr` GitHub remote listed under `Remotes:` in `DESCRIPTION`.
- The `manifest.json` at the project root is used by Connect to resolve the app entry point (`app.R`).
- Report generation (Word `.docx`) requires Quarto to be available on the Connect server, as the templates in `inst/markdown/` are `.qmd` files rendered via `rmarkdown::render()`.
